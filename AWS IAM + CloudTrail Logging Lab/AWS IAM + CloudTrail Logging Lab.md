🎯PROJECT OVERVIEW

This project demonstrates secure **IAM user management** and **activity logging** in AWS.

This setup reflects real-world cloud security practices used for **monitoring, auditing, and compliance**.

### **Step 3: Enable CloudTrail (AWS Logging Service)**

1. Go to **CloudTrail → Create trail  → Under Trail Details → Click on “…Create Trail”** as highlighted in the screenshot.

- Log file SSE-KMS encryption: Tick this (this serves as encryption for the S3 Bucket)
- Customer managed AWS KMS key: Choose New and enter a unique alias name (paul-7115-trails)
- Under Cloud Watch Logs Optional: Tick the Enabled under CloudWatch Logs
- Under Log Group Name: i renamed mine to “**aws-cloudtrail-logs-my-personal-trails”**

- For Log Events: Under Events- Tick only Management Events; Under Management Events → Under API Activity → ✅ Read & ✅Write ONLY.

### Step 6: Using CloudWatch to View CloudTrail Logs

**CloudWatch** is an AWS (Amazon Web Services) monitoring and observability service.

From the AWS Search Bar, search for CloudWatch click on it, then Navigate to Logs and choose Logs Insight.

1. Under Log Groups, Click on the empty space and select your cloudtrails na,me as seen below and make sure it’s ticked.

![Under Log Groups, Click on the empty space and select your cloudtrails name as seen below and make sure it’s ticked.](<screenshot/image%2033.png>)

Under Log Groups, Click on the empty space and select your cloudtrails name as seen below and make sure it’s ticked.

1. Next Step is to click on the Query Generator fill out the prompt on what query you desire to generate, from my cloudtrail events, i created an IAM User and Security Group with Inbound and Outbound rules, so i used this prompt **“Generate a Query that shows the IAM User i created, and the Security Group I created with inbound and outbound rules from CloudTrail”**

1. Then I adjusted the interval to reflect the date and time range i created these events, i applied the settings 

![Adjust the interval to reflect the date and time range i created these events, and click on Apply](<screenshot/image%2034.png>)

Adjust the interval to reflect the date and time range i created these events, and click on Apply

1. Next Step was to click on Generate New Query, then Run Query, as you can see the query was inputted was generated, then we can scroll down and see the logs 

![ Click on Generate New Query, then Run Query](<screenshot/image%2035.png>)

 Click on Generate New Query, then Run Query

**Generated Query:**

fields eventTime, eventName, requestParameters.userName, requestParameters.groupName, userIdentity.arn 
| filter eventName in ["CreateUser", "CreateSecurityGroup", "AuthorizeSecurityGroupIngress", "AuthorizeSecurityGroupEgress"]
| sort eventTime desc

These are the captured events that we performed, it shows the Creation of the IAM User (BlackBerry) and the Creation of the Security Group (CloudTrail-Test User), we can drill down to see more information from each log event.

| sort *@timestamp* desc
