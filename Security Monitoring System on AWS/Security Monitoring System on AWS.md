### **STEP 3: Create CloudWatch Alarm & Link to SNS Topic.**

It’s important to note that when creating a CloudWatch Alarm, Metrics only appear after data is emitted: 

CloudWatch won't list a custom metric until at least one matching log event has triggered the filter. (https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/viewing_metrics_with_cloudwatch.html?utm_source=chatgpt.com)

This implies that when trying to create an Alarm, you will not see the Custom Metric Namespace that was created earlier.

In order to resolve this, I triggered an event using the AWS CLI by running this command below:

```
aws secretsmanager get-secret-value --secret-id Production_Database_Credentials
aws cloudwatch list-metrics --namespace "MySecrets"
```

After that I followed these steps to Create the CloudWatch Alarm

1. Navigate to AWS Console Search for → CloudWatch → **Alarms** → **Create Alarm**.
2. **Select Metric**
3. Click **Select metric** → **Custom Namespaces** → find “MySecrets”, then click on “**Metrics with no dimensions”**
4. Tick the metric name called: “**MySecretTriggers**” then Click **Select metric**.
5. **Configure Alarm**
    - Metric Name: **MySecretTriggers**
    - **Statistic:** Sum
    - **Period:** 1 minute
    - **Threshold type:** Static
    - **Whenever “MySecretTriggers is…”:** >=1 (Greater/Equal then type 1 inside the define threshold value box)
    - Alarm State Trigger: Choose In Alarm
    - Send a notification to the following SNS topic: Tick Select an existing SNS topic
    - **Send a notification to…: Click the box and choose SecretsTrigger (our SNS Topic Name)**
    - Ignore the other settings and click Next
6. **Name Alarm:** **Prod-Secrets-Alarm**
7. Optional description: **This Alarm is to inform me about access to my AWS Secrets,** then click Next to Preview settings and Click **Create Alarm.**

### **Phase 4: The “Active Defense” Kill-Switch (Using Lambda)**

I will be creating a simple **AWS Lambda** function (using Python or Node.js) that uses the AdminDetachUserPolicy or PutUserPermissionsBoundary API call to strip all permissions from a user.

### Step 1: Create Lambda Function on AWS Console

1. AWS Console → Navigate to Lambda → Click Create a Function
2. Choose: Author from scratch
3. Name: **SecretsKillSwitch**
4. Runtime: Python 3.14
5. Architecture: x86_64 (default)
6. Click Create Function

![Lambda Function Created Successfully.](image%2016.png)

Lambda Function Created Successfully.

So I will create a python code that applies AWSDenial of Policy to anyone (IAM User) who tries to access the Secrets, which means the User will not have access to the Secret we created earlier.

The Code Below will be copied and pasted inside the visual editor under the Code → Code Source, Then Click on Deploy

```
import boto3
def lambda_handler(event, context):
iam = boto3.client('iam')
username = event['detail']['userIdentity']['userName']
iam.attach_user_policy(UserName=username, PolicyArn='arn:aws:iam::aws:policy/AWSDenyAll')
return {'status': 'Neutralized', 'user': username}
```
